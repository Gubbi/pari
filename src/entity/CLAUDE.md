## What Does Not Live Here

- Caller-facing API shaping, viewer/editor handles, validation rules → `workspace`
- Dispatch surfaces, message flow, checkout lifecycle, JSON ↔ tracked conversion → `store`
- Persistence layout, asset schemas, backends → `substrate`
- Cross-cutting error classification → `error`

If an edit starts to discuss store dispatch, asset writes, or rule execution order, it probably belongs in another layer.

## Conventions Worth Repeating Locally

- Top-level ref: `EntityRef::new(id)`. Embedded ref: `EntityRef::with_parent(id, parent)`.
- Parent is part of semantic identity — two embedded entities with the same id under different parents are distinct. Do not reintroduce workflow-id-only helpers.
- `EntityRef<T, P>::to_any_ref(&self)` is an instance method. Generated by `#[derive(Entity)]`.
- Tracked field paths: `TrackedField::loaded(value)` for the JSON-to-tracked pipeline (load and insert paths in the store), `Arc::new(TrackedField::mutated(value))` for COW setter replacement on the editor side.
- `Tracked<Name>` carries only state and `pub(crate)` helpers — no public constructor, no async accessors. Construction goes through the store's JSON-to-tracked pipeline; `TrackedEntity::from_json_value` is `pub(crate)` and reachable only there.
- `Step` is not an entity — no `EntityRef`, no `#[derive(Entity)]`.
- `#[derive(Entity)]` generates behavior for multiple layers (`entity` for `Tracked<Name>` + `impl Entity`, `workspace` for `XViewer<'ws, Name>` and `XEditor<'ws, Name>`). Ownership of the generated items follows the consuming layer, not the macro crate.
- The per-entity `XViewer<'ws, Name>` (workspace-owned) is generated alongside `Tracked<Name>` (entity-owned). Read accessors and `validate` / `validate_with` live on the viewer; setters and `commit(self)` / `undo_checkout(self)` live on `XEditor<'ws, Name>` which derefs to the viewer. The `Entity` trait's `take` / `into_tracked_entity` methods round-trip the tracked form through editor construction inside `Workspace::checkout`.
- New entity fields cannot use struct-keyed maps — the substrate intermediate is `serde_json::Value`, which only allows string keys. Use `Vec<(K, V)>` plus a uniqueness validation rule. See `docs/design/layers/entities.md` *Authoring Constraints*.
