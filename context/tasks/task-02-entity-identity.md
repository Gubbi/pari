# Task 02 — Entity Identity Infrastructure

## Scope

Define the core identity and trait infrastructure that every entity type builds on:

- `ParentKind` trait + `NoParent` + `WorkflowParent`
- `EntityRef<T, P>` — typed entity reference (id + optional parent chain)
- `Entity` trait — KIND, validation_schema(), Parent, Tracked
- `TrackedEntity` companion trait — roundtrip `TrackedX::Entity = X`
- `Tracked<T>` type alias — ergonomic `Tracked<Role>` instead of `<Role as Entity>::Tracked`
- `EntityKind` enum — hand-written here; will be replaced by `entity_registry!` output in Task 04
- `AnyEntityRef` enum — hand-written here; will be replaced by `entity_registry!` output in Task 04
- `Resolvable` trait — `to_any_ref` + `extract`; generated by `#[derive(Entity)]`, declared here

## Design Note: Naming Collision

The design names both the companion trait and the store-level enum `TrackedEntity`. These cannot coexist in the same namespace. Resolution: the companion trait is named `TrackedEntity`; the store-level enum (introduced in Task 09) is named `StoreEntity`. This is the only place this deviation is noted — it applies throughout the implementation.

## Files

- `src/entity.rs` — new file; all types listed above
- `src/lib.rs` — add `pub mod entity;`

### `src/entity.rs`

use std::marker::PhantomData;

// ---------------------------------------------------------------------------
// ValidationSchema — placeholder type; filled out fully in Task 06
// ---------------------------------------------------------------------------

/// Placeholder. Task 06 replaces this with the full ValidationSchema<E> type.
pub struct ValidationSchema<E>(std::marker::PhantomData<E>);

// ---------------------------------------------------------------------------
// ParentKind
// ---------------------------------------------------------------------------

/// Marker trait for parent type parameters on EntityRef.
pub trait ParentKind: private::Sealed {}

mod private {
    pub trait Sealed {}
}

/// Top-level entities have no parent.
pub struct NoParent;
impl private::Sealed for NoParent {}
impl ParentKind for NoParent {}

/// Embedded entities (Task, Relay, EmbeddedWorkflow) live inside a workflow.
pub struct WorkflowParent {
    /// The id of the containing workflow.
    pub workflow_id: String,
}
impl private::Sealed for WorkflowParent {}
impl ParentKind for WorkflowParent {}

// ---------------------------------------------------------------------------
// EntityRef<T, P>
// ---------------------------------------------------------------------------

/// A typed reference to an entity: its string id and optional parent context.
///
/// `T: Entity` ensures only declared entity types can form refs.
/// `P = NoParent` default covers top-level entities.
///
/// Hash and Eq are derived from (KIND, id, parent_id) so refs are usable
/// as HashMap keys.
pub struct EntityRef<T: Entity, P: ParentKind = NoParent> {
    pub(crate) id: String,
    pub(crate) parent: P,
    _marker: PhantomData<T>,
}

impl<T: Entity> EntityRef<T, NoParent> {
    /// Construct a top-level entity reference.
    pub fn new(id: impl Into<String>) -> Self;
}

impl<T: Entity> EntityRef<T, WorkflowParent> {
    /// Construct an embedded entity reference with its parent workflow id.
    pub fn new_embedded(id: impl Into<String>, workflow_id: impl Into<String>) -> Self;
}

impl<T: Entity, P: ParentKind> EntityRef<T, P> {
    pub fn id(&self) -> &str;
}

// Hash, Eq, Clone, Debug impls — keyed on (T::KIND discriminant, id, parent id if any)
impl<T: Entity, P: ParentKind + PartialEq> PartialEq for EntityRef<T, P> { ... }
impl<T: Entity, P: ParentKind + Eq> Eq for EntityRef<T, P> { ... }
impl<T: Entity, P: ParentKind + Hash> Hash for EntityRef<T, P> { ... }
impl<T: Entity, P: ParentKind + Clone> Clone for EntityRef<T, P> { ... }

// PartialEq / Eq / Hash / Clone for NoParent and WorkflowParent:
// NoParent: all instances are equal; hash is constant.
// WorkflowParent: equality and hash based on workflow_id.

// ---------------------------------------------------------------------------
// Entity trait
// ---------------------------------------------------------------------------

pub trait Entity: Sized + 'static {
    const KIND: EntityKind;
    fn validation_schema() -> &'static ValidationSchema<Self>;
    type Parent: ParentKind;
    type Tracked: TrackedEntity<Entity = Self>;
}

// ---------------------------------------------------------------------------
// TrackedEntity companion trait
// ---------------------------------------------------------------------------

/// Implemented by every tracked entity struct.
/// Provides the roundtrip: `TrackedRole::Entity = Role`
/// and therefore `Role::Tracked::Entity = Role`.
pub trait TrackedEntity {
    type Entity: Entity<Tracked = Self>;
}

// ---------------------------------------------------------------------------
// Tracked<T> type alias
// ---------------------------------------------------------------------------

/// Ergonomic form of `<T as Entity>::Tracked`.
/// `Tracked<Role>` instead of `<Role as Entity>::Tracked`.
pub type Tracked<T> = <T as Entity>::Tracked;

// ---------------------------------------------------------------------------
// EntityKind — hand-written; Task 04 replaces with entity_registry! output
// ---------------------------------------------------------------------------

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum EntityKind {
    Role,
    Hook,
    Team,
    Workflow,
    ReusableWorkflow,
    ArtifactKind,
    Task,
    Relay,
    EmbeddedWorkflow,
}

// ---------------------------------------------------------------------------
// AnyEntityRef — hand-written; Task 04 replaces with entity_registry! output
// ---------------------------------------------------------------------------
// Forward declarations only — concrete variants require the entity types
// to be defined first (Task 05). See Task 04 for the full generated enum.

// ---------------------------------------------------------------------------
// Resolvable — declared here, implemented by #[derive(Entity)] (Task 03)
// ---------------------------------------------------------------------------

/// Enables generic `Store::resolve<T>()` without per-type methods.
/// Both methods are generated by `#[derive(Entity)]`.
pub trait Resolvable: Entity {
    /// Convert a typed EntityRef to the type-erased AnyEntityRef.
    fn to_any_ref(entity_ref: &EntityRef<Self, Self::Parent>) -> AnyEntityRef;

    /// Pattern-match the StoreEntity enum to return this entity's tracked form.
    /// Returns None if the variant doesn't match (indicates a store invariant violation).
    fn extract(entity: &StoreEntity) -> Option<&Self::Tracked>;
}

## Implementation Notes

- `NoParent`: implement `PartialEq`, `Eq`, `Hash`, `Clone`, `Copy` — all trivial (no fields).
- `WorkflowParent`: implement those traits delegating to `workflow_id: String`.
- `EntityRef` stores `id: String`, `parent: P`, `_marker: PhantomData<fn() -> T>` (use `fn() -> T` to keep the marker covariant/correct for lifetimes).
- Hash for `EntityRef` must incorporate `T::KIND` to prevent different entity types with the same id from colliding.
- `AnyEntityRef` and `StoreEntity` stubs: use empty enums with a comment `// Filled in by entity_registry! in Task 04`.
