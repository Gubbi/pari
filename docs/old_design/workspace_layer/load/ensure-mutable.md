# ensure-mutable

**Owning layer: `store`**

---

## Purpose

Every generated workspace-layer `set_*()` accessor is async and sends `StoreRequest::EnsureMutable` to `EntityServer` before mutating the field. `EntityServer` owns the preparation flow and ensures the field is ready for mutation by loading prerequisites and the field itself if required. This prevents writes to a stub that would be silently overwritten on the next load.

---

## Call Pattern

Generated by `#[derive(Entity)]` inside each `set_*()` setter:

```rust
pub async fn set_name(&mut self, value: String) -> Result<(), SetterError> {
    EntityClient::ensure_mutable(self.entity_ref.to_any(), "name").await?;
    self.name = Arc::new(TrackedField::mutated(value));
    Ok(())
}
```

The setter sends a single `StoreRequest::EnsureMutable { any_ref, field }` message and awaits completion. If `EntityServer` triggers any loads as a result, it initializes the store's Arcs in-place — the caller's cloned `Arc<TrackedField<T>>` for those fields is the same allocation, so the writes are immediately visible on the caller's copy. Once `ensure_mutable` returns `Ok`, the mutation proceeds.

---

## EntityServer: EnsureMutable Steps

`EntityServer` handles `StoreRequest::EnsureMutable { any_ref, field }` as follows:

1. Look up `load_strategy(kind, field)` — internal to `EntityServer`; callers never access it directly.

2. Load prerequisites unconditionally:
   - If `load_strategy.prerequisites` is non-empty, call `load(any_ref, prerequisites)` for any uninitialized prerequisite fields.
   - This runs regardless of `mutable_without_load`.

3. If `load_strategy.mutable_without_load == false`:
   - Call `load(any_ref, &[field])` to load the field itself before allowing mutation.

4. Return `Ok(())` — the field is now safe to overwrite.

Step 1 (`load_strategy`) is internal to the `store` layer's `EntityServer` in collaboration with the `substrate` layer. The `workspace` layer knows only that a field must be prepared for mutation — it does not know about assets, file layout, or which other fields share storage with the mutated field.

---

## When `mutable_without_load` is `true`

For substrates with `supports_partial: true`, individual fields can be written without reading the full asset — no field pre-load is needed. However, prerequisites declared in `load_strategy.prerequisites` still load unconditionally (step 2). A field with no prerequisites and `mutable_without_load: true` can be set on a stub entity directly.

`mutable_without_load: true` also applies on `RepoSubstrate` for single-field assets — assets where the written field is the only field in the file. A full-file overwrite requires no prior read. Prerequisites still apply for path resolution.

---

## Enforcement at Mutation Time

The check runs at mutation time (`set_*()`) — not at persist time. By the time `persist` runs, any mutated field is guaranteed to have been prepared.
