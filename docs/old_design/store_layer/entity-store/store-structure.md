# store-structure

**Owning layer: `store`**

---

## Purpose

`Store<S>` is the central data structure owned by `EntityServer`. It holds all loaded entities in a single flat map keyed by `AnyEntityRef`, tracks which entities have changed since the last persist, and holds the substrate reference.

---

## Structure

```rust
struct Store<S: Substrate> {
    entities:  HashMap<AnyEntityRef, TrackedEntity>,
    added:     HashSet<AnyEntityRef>,
    modified:  HashSet<AnyEntityRef>,
    removed:   HashSet<AnyEntityRef>,
    checked_out: HashSet<AnyEntityRef>,
    substrate: S,
}
```

---

## TrackedEntity

One variant per entity kind. Defined alongside `AnyEntityRef` — both grow together when a new entity kind is added.

```rust
enum TrackedEntity {
    Role(TrackedRole),
    Hook(TrackedHook),
    Team(TrackedTeam),
    Workflow(TrackedWorkflow),
    ReusableWorkflow(TrackedReusableWorkflow),
    ArtifactKind(TrackedArtifactKind),
    Task(TrackedTask),
    Relay(TrackedRelay),
    EmbeddedWorkflow(TrackedEmbeddedWorkflow),
}
```

---

## Resolvable Trait

Generated by the derive macro for each entity type. Enables a single generic `resolve<T>()` instead of per-type methods.

```rust
trait Resolvable: Entity {
    fn to_any_ref(entity_ref: &EntityRef<Self, Self::Parent>) -> AnyEntityRef;
    fn extract(entity: &TrackedEntity) -> Option<&Self::Tracked>;
}
```

```rust
fn resolve<T: Resolvable>(&self, entity_ref: &EntityRef<T, T::Parent>)
    -> Result<&T::Tracked, ResolveError>
{
    let any_ref = T::to_any_ref(entity_ref);
    let entity = self.entities.get(&any_ref).ok_or(ResolveError::NotFound)?;
    Ok(T::extract(entity).expect("store invariant: kind mismatch impossible"))
}
```

A kind mismatch between key and value is impossible by construction — `insert` always pairs an `AnyEntityRef` with its corresponding `TrackedEntity` variant.

---

## insert

```rust
fn insert<T: Resolvable>(&mut self, entity: T, entity_ref: EntityRef<T, T::Parent>)
```

Converts the plain entity to a tracked entity via `From<T>`, wraps it in the appropriate `TrackedEntity` variant, inserts into `entities`, and adds the ref to `added`. New-then-remove and remove-then-new transition logic is applied — see [62 · entity-change-lists](../change-tracking/entity-change-lists.md).

---

## remove

```rust
fn remove(&mut self, any_ref: &AnyEntityRef) -> TrackedEntity
```

Evicts from `entities` and returns the evicted value. Applies transition logic before updating `removed` — see [62 · entity-change-lists](../change-tracking/entity-change-lists.md).

The caller may call `insert()` with the returned value to undo the removal.

---

## undo_commit

```rust
fn undo_commit(&mut self, any_ref: &AnyEntityRef) -> Result<(), UndoError>
```

Reverts to last persisted state. If `added`: evicts from `entities` and `added`. If `modified`: replaces entity with a fresh stub and removes from `modified`. Returns `Err(WrongState)` if entity is not in `added` or `modified`, or is checked out.

---

## unload

```rust
fn unload(&mut self, any_ref: &AnyEntityRef) -> Result<(), UndoError>
```

Replaces a clean entity with a fresh stub. Returns `Err(WrongState)` if entity is not in clean state (i.e., is in `added`, `modified`, `removed`, or `checked_out`).

---

## checkout overlay

`checked_out` is not a primary state — it is an independent overlay that can coexist with `added`, `modified`, or `clean`. An entity in `modified + checked_out` is simultaneously tracked as a pending change and locked for exclusive mutation.

The primary states (`added`, `modified`, `removed`) are mutually exclusive. `checked_out` is orthogonal to all of them. See [48b · store-entity-lifecycle](store-entity-lifecycle.md) for the full state machine and operation preconditions.
