# tracked-role

**Data Model → `data_model/tracked-entity/`**

---

## Purpose

`TrackedRole` is the store's mutable, load-aware representation of a `Role`. All fields are wrapped in `Arc<TrackedField<T>>` for cheap COW cloning. No back-reference to the store — communication goes through `EntityServer::sender()`.

---

## Structure

```rust
struct TrackedRole {
    entity_ref:  EntityRef<Role>,
    name:        Arc<TrackedField<String>>,
    description: Arc<TrackedField<Option<String>>>,
    purpose:     Arc<TrackedField<String>>,
    traits:      Arc<TrackedField<Option<Vec<String>>>>,
    extensions:  Arc<TrackedField<Extensions>>,
}
```

`description` and `traits` are `TrackedField<Option<_>>` — OnceLock initialized with `None` means loaded and confirmed absent.

## Setters

```rust
async fn set_name(&mut self, val: String)               -> Result<(), SetterError>
async fn set_description(&mut self, val: Option<String>) -> Result<(), SetterError>
async fn set_purpose(&mut self, val: String)            -> Result<(), SetterError>
async fn set_traits(&mut self, val: Option<Vec<String>>) -> Result<(), SetterError>
async fn set_extensions(&mut self, val: Extensions)     -> Result<(), SetterError>
```

Each setter calls `ensure_mutable` first, then runs structural and entity-local semantic validation on the incoming value, then COW-replaces the field's `Arc<TrackedField<T>>` with a new `TrackedField` whose `OnceLock` is pre-seeded with the new value and `dirty = true`. A validation failure returns `SetterError::Validation` without modifying the field.

`Role` has no cross-entity refs — setters never need to call `has_ref`. See [82 · validation-api](../../validation/validation-api.md) for `SetterError` definition.

---

## ensure_mutable

Loads **all** fields before any mutation. `RepoSubstrate` writes the full entity file at persist — a partial field snapshot would produce a corrupt file. Substrate granularity (full file per entity) drives what must be pre-loaded, not cross-entity dependencies.

Role has no cross-entity references, so `ensure_mutable` sends a single `StoreRequest::Load` for all fields via `EntityServer::sender()`.
