# all-refs

**Entity Layer → `entity_layer/tracked-entity/`**

---

## Purpose

`all_refs()` collects all cross-entity references held as field values on a tracked entity. It is used inside `EntityServer`'s load handler as a pre-fetch step — after loading the requested fields, `EntityServer`'s load handler calls `all_refs()` and batch-resolves all referenced entities from the substrate in one call.

The validator's `has_ref` checks will resolve these refs individually anyway — `all_refs()` ensures they are already in the store by the time validation runs, avoiding N separate substrate fetches.

---

## Signature

```rust
fn all_refs(&self) -> Vec<AnyEntityRef>
```

Generated by the derive macro. Iterates over all `Arc<TrackedField<EntityRef<T>>>` and `Arc<TrackedField<Option<EntityRef<T>>>>` fields. If the field's `OnceLock` is initialized, the ref is converted to `AnyEntityRef` and included. Uninitialized fields are skipped.

---

## Examples

```rust
TrackedRole::all_refs()   // → []  (no cross-entity fields)
TrackedRelay::all_refs()  // → [AnyEntityRef::ReusableWorkflow(...)]  if delegates_to is loaded
```

---

## Usage in EntityServer's Load Handler

`EntityServer`'s load handler algorithm:

1. Get prerequisites for the requested fields
2. Fetch prerequisites from substrate
3. Load the requested fields
4. Call `entity.all_refs()` on the now-populated entity
5. Batch-resolve all returned refs from the substrate in a single call

No message-passing — this runs entirely within EntityServer.
