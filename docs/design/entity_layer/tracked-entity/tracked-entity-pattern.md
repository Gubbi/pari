# tracked-entity-pattern

**Owning layer: `entity`**

---

## Purpose

A tracked entity is the store's mutable, load-aware representation of a plain entity. It wraps each field in `Arc<TrackedField<T>>` for cheap cloning and COW mutation. There is no back-reference to the store — entities communicate with the server via `EntityServer::sender()` directly.

---

## Anatomy

```rust
struct TrackedRole {
    entity_ref: EntityRef<Role>,
    name:       Arc<TrackedField<String>>,
    purpose:    Arc<TrackedField<String>>,
    traits:     Arc<TrackedField<Option<Vec<String>>>>,
    extensions: Arc<TrackedField<Extensions>>,
}
```

Each domain field is `Arc<TrackedField<T>>`. `TrackedField<T>` uses `std::sync::OnceLock<T>` internally — uninitialized means not yet loaded; initialized means loaded. Optional domain fields use `TrackedField<Option<T>>` — initialized with `None` means loaded and confirmed absent. No store back-reference — `EntityServer::sender()` is the global access point.

---

## New / Modified / Removed Tracking

Tracked entities carry no version or dirty flag at the entity level. New, modified, and removed status is tracked by the store via `added`, `modified`, `removed` `HashSet<AnyEntityRef>` — see [62 · entity-change-lists](../../store_layer/change-tracking/entity-change-lists.md). The entity is only responsible for field-level dirty state via `TrackedField.dirty`.

---

## Load States (Emergent)

Not explicit enum variants — they emerge from field state:

| State | Meaning |
|---|---|
| Stub | All fields uninitialized — entity_ref only; entity known to exist, nothing loaded |
| Partial | Some fields initialized, others uninitialized |
| Full | All fields initialized |

---

## Accessors

```rust
async fn name(&self) -> Result<&str, LoadError>
```

Generated by the derive macro. If the field's `OnceLock` is already initialized, returns the value immediately; if uninitialized, sends a `StoreRequest::Load` via `EntityServer::sender()`, awaits the result, calls `OnceLock::set()` (silently discarding if another task initialized it concurrently), then returns the value. Callers always receive a loaded value.

---

## Setters

```rust
async fn set_name(val: String) -> Result<(), SetterError>

enum SetterError {
    Substrate(SubstrateError),    // from ensure_mutable
    Validation(ValidationErrors), // from structural + entity-local semantic validation
}
```

Each setter:
1. Calls `ensure_mutable` — loads prerequisite fields; may return `SubstrateError`
2. Runs structural (#1) and entity-local semantic (#2) validation on the incoming value
3. On validation failure: returns `SetterError::Validation`; field is not modified
4. On success: COW-replaces the field's `Arc<TrackedField<T>>` with a new `TrackedField` whose `OnceLock` is pre-seeded with the new value and `dirty = true`

Cross-entity validation (#3) is not run in setters — it requires store access and runs at load-time and check-in. See [82 · validation-api](../../validation_layer/validation-api.md).

---

## ensure_mutable

Called inside setters. Loads pre-requisite fields in-place — updates the entity's `Arc<TrackedField<T>>` directly without replacing the whole entity. Uses `EntityServer::sender()` to send a `StoreRequest::Load`.
