# ensure-mutable

**Workspace Layer → `workspace_layer/load/`**

---

## Purpose

`ensure_mutable` is a pre-mutation check called inside every generated `set_*()` accessor. It guarantees the containing asset is loaded before a field is mutated — preventing writes to a stub that would be silently overwritten on the next load.

---

## Unit of Mutation Safety: The Asset

Mutation safety is enforced at the **asset** level, not the field level. An asset is a cohesive storage unit (e.g., a markdown file). For substrates that write full assets (RepoSubstrate), mutating any field in an asset requires all fields in that asset to be loaded first — otherwise the write would overwrite unloaded fields with empty values.

---

## Call Pattern

Generated by `#[derive(Entity)]` inside `set_*()`:

```rust
pub fn set_name(&mut self, value: String) {
    self.ensure_mutable("name");
    let lock = OnceLock::new();
    let _ = lock.set(value);
    self.name = Arc::new(TrackedField { value: lock, dirty: true });
}

fn ensure_mutable(&mut self, field: &str) {
    let strategy = S::load_strategy(Self::KIND, field);
    if !strategy.mutable_without_load {
        // Load the containing asset's fields and merge them into self.
        // This keeps the client-owned TrackedEntity consistent — at commit,
        // merge_dirty_into sends back exactly the entity the caller mutated.
        let loaded = EntityServer::sender().load(self.entity_ref.to_any(), &[field]);
        self.merge_non_dirty(&loaded);  // initializes uninitialized OnceLock fields; never overwrites initialized
    }
}
```

`ensure_mutable` is `&mut self`. The load result is merged into the client-owned entity before the mutation proceeds, so the entity the caller holds and eventually commits reflects all loaded fields — not just the one being set.

---

## Multi-Asset Entities

For entities with more than one asset, `ensure_mutable` loads the asset containing the mutated field, plus any prerequisite fields declared in `load_strategy.prerequisites`. Prerequisites may live in a different asset — they must be loaded first because the mutated field's asset path may depend on them.

```
// Hypothetical: mutating a field in a single-field asset whose path depends on "category"
Entity: mutating "attachment" (in asset; path depends on field "category")
  → load_strategy("attachment") → prerequisites: ["category"], mutable_without_load: true
  → "category" not in self → load ref_asset first → merge into self
  → path resolved using category value; attachment content not loaded (single-field, full overwrite)
  → attachment is now mutable

// Mutating a field in the ref_asset (multi-field, full-file rewrite)
Entity: mutating "name" (in ref_asset)
  → load_strategy("name") → prerequisites: [], mutable_without_load: false
  → load ref_asset fields → merge into self
  → other asset not loaded (different asset, no prerequisite declared)
```

---

## When `mutable_without_load` is `true`

For substrates with `supports_partial: true`, individual fields can be written without reading the full asset — no asset-level pre-load is needed. However, `ensure_mutable` still loads any prerequisites declared in `load_strategy.prerequisites`. A field with no prerequisites and `mutable_without_load: true` can be set on a stub entity directly.

`mutable_without_load` can also be `true` on RepoSubstrate for single-field assets — assets where the field being written is the only field in the file. Writing such an asset is always a full-file overwrite and requires no prior read. Prerequisites declared in `load_strategy.prerequisites` still apply — they handle path resolution, not content loading.

For example, a single-field asset whose path includes `{category}` has `mutable_without_load: true`. The `category` prerequisite is still required to resolve the path — but the file's existing content is never loaded before the write.

---

## Enforcement at Mutation Time

The check runs at mutation time (`set_*()`) — not at persist time. This keeps the write pipeline simple: by the time `persist` runs, any mutated asset is guaranteed to be fully loaded.
