# entity-kind-naming

**Codegen → `codegen/`**

---

## Purpose

`EntityKind` and `Entity` address the same concept at two different levels. This doc names each one unambiguously to prevent confusion when reading code and design docs.

---

## The Two Concepts

### 1 · `EntityKind` — runtime enum

```rust
pub enum EntityKind {
    Role,
    Hook,
    Team,
    ArtifactKind,
    Workflow,
    ReusableWorkflow,
    EmbeddedWorkflow,
    Task,
    Relay,
}
```

A runtime value. Used wherever entity type information must survive past monomorphization:

- **`EntityRef` hash/eq** — `T::KIND` is hashed so refs to different entity types with the same id never collide
- **`AnyEntityRef` dispatch** — the `kind()` accessor delegates back to the stored `EntityKind` variant
- **Substrate `load_strategy(entity_kind, field)`** — passed as a function argument to select the correct schema
- **Serde wire format** — carried as the `kind` tag in serialized `EntityRef` objects

---

### 2 · `Entity` — compile-time trait

```rust
pub trait Entity {
    const KIND: EntityKind;
}
```

A compile-time bound. Every entity struct type implements `Entity`, generated by `#[derive(Entity)]`.

`const KIND` provides the `EntityKind` variant as a compile-time constant. Generic code (`EntityRef<T: Entity>`, store methods) accesses it via `T::KIND` — monomorphized away at compile time, no runtime dispatch.

---

## How They Relate

`T::KIND` and `EntityKind::T` are the same value — `const KIND` is just the way to read it in a generic context:

```rust
// In a generic function:
fn foo<T: Entity>() { let k: EntityKind = T::KIND; }

// At the call site, T::KIND is exactly:
foo::<Role>()    // T::KIND == EntityKind::Role
foo::<Task>()    // T::KIND == EntityKind::Task
```

The trait exists because entity types need a compile-time contract for `EntityRef<T: Entity>`. The enum exists because some APIs receive entity kind as a runtime argument.

---

## `AnyEntityRef` — type erasure without `dyn`

`AnyEntityRef` (see [12 · any-entity-ref](../entity_layer/entity-identity/any-entity-ref.md)) is the concrete enum that provides type erasure without trait objects:

```rust
pub enum AnyEntityRef {
    Role(EntityRef<Role, NoParent>),
    // ...
}
```

`kind()` on `AnyEntityRef` returns the `EntityKind` variant by matching on the enum. No `dyn` trait, no vtable. The derive macro generates `Resolvable::to_any_ref()` to convert typed `EntityRef<T, P>` into the correct `AnyEntityRef` variant.
